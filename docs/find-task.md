【当前问题】
我的漫画文件在 Android 手机上，用户通过系统目录授权（SAF / Document Tree）选择漫画库目录后，当前实现存在这些问题：

1. 如果授权的是漫画库根目录，或者授权到“漫画名”这一层目录，扫描会很慢，图片多时甚至会卡死。
2. 当前实现似乎把“用户授权的目录”直接当成“漫画正文图片目录”处理了，没有正确区分：
   - 授权目录
   - 漫画目录
   - 真正存放图片的最末级目录（正文图片目录）
3. 对于这种结构：
   漫画库/漫画名/*.png
   可以正常理解。
4. 但对于这种结构：
   漫画库/漫画名/1192423/*.png
   当前实现容易出问题：
   - 如果授权到漫画名目录，扫描很慢甚至卡死
   - 如果授权到 1192423 目录，读取正常，但漫画标题会错误地显示为“1192423”
5. 我希望支持多种本地漫画目录格式，并让用户只需要授权“漫画库根目录”即可，应用自己自动识别。

1. 单层图片目录：

   漫画库/漫画名/1192423/0.png

- 单层图片目录：封面 = 当前目录第一张图
- 双层图片目录：封面 = 子目录第一张图
